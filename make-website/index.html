How to Set Up Your Princeton Website!

(www.princeton.edu/~netid)

A few people have asked me how to go about setting up their own through Princeton -- OIT’s resources are a bit hard to parse through for this topic, so I figured it would be worth it to write a piece on making a Princeton-hosted website!

How it Works

Princeton allocates 5gb of space to everyone (unless you’re class of 2017+, in which case you get a tenth that). This is your H: Drive, located on the Central File Server. These are all the files you see when you log into a cluster computer on campus. Inside this, there’s a special file called your public_html folder, which OIT has set up to link . Anything you put in there will accessible by anyone on the internet!

The Process

  1. First, let’s make sure you can connect to the Central File Server. If you’re on campus network, you’ve got access! If not, get on it by setting up a VPN connection by following instructions for Mac or Windows.

    Note: according to OIT, VPN will very soon be retired and replaced with Secure Remote Access (SRA).

  2. Next, let’s get to your public_html folder!

    • Mac

      For Mac, we connect to our public_html folder directly. (Warning: don't connect to your H: drive!) The simplest way to connect from a Mac is to connect through Finder. Follow the instructions below or in a little more detail here, from OIT here.

      Open Finder, click Go in the top menu, and click "Connect to Server"

      In the dialog that pops up, enter "smb://netid@files.princeton.edu/home/public_html/netid"

    • Windows

      For Windows, we connect to our H: Drive and then open the public_html folder within. The instructions from OIT are pretty clear -- follow them here.

      Once you're in, open your public_html folder--you're in!

    • SSH (advanced)

      If you're familiar with using commands like ssh and scp, you can easily ssh into your H: Drive. Make sure your Unix account is enabled by going here, and then ssh into netid@arizona.princeton.edu. From there, scp your files into the public_html folder!

      Note that ssh is more secure, but that doesn't really matter when we're talking about a public site, right?

  3. Now, you should be looking at a (probably empty) folder of your public_html folder. Anything you put in it will appear on the world wide web! If you put a file in your public_html folder, it will appear at www.princeton.edu/~netid/filename. For example, if I put the file "adorable_kitten.jpg" in my folder, you can see it here!

    Try it with a file and make sure it works!

  4. Next, let's make this a working website. We've seen how to go to the url of a specific file -- but what happens when someone goes to www.princeton.edu/~netid without specifying a filename (basically, the whole folder)? Their browser will look for a file called "index.html". If you make an index file and stick it in your folder, that'll be your homepage!

    Here's an example of a sample index file.

    <html>
<head>
  <title>Sample Homepage!</title>
</head>
<body>

  <h1>Welcome to my website!</h1>

  <p>Thanks for visiting my page! I'm a student and a pretty cool person. I once fought off a bear to save a crying baby! I also have purple hair.</p>

  <p>Here are some links to things:
  <ul>
    <li><a href="http://www.princeton.edu/website.html">Here</a>'s the tutorial I got this from!
    <li>Look at this cute <a href="adorable_kitten.jpg">kitten</a>.
  </ul>

</body>
</html>

    Copy and paste this into a text file, save it as "index.html" in your folder (beware: not "index.html.txt"), and go to www.princeton.edu/~netid. You should see this!

    To understand what's going on, it's best to learn some basic html, which is the language all this is written in. Here's a good place to start learning! A great way to learn is to try replacing the text I wrote with your own. If something doesn't work, try again! A key thing to look it is that text in the href="_____" bit -- that's a link! Notice I can link to things in the public_html folder just by typing the filename.

    A really easy way to make a nice website is to google a few templates, or learn about css. To easily make your website better, I recommend Bootstrap

    Also, you'll notice that the website starts out in Times New Roman.. how boring! To add a little spice, we can style our text differently. Here's an example of a little more colorful page, with an image. This is what it should look like (unfortunately, it still looks like a 90s Geocities website). 

    <html>
<head>
  <title>Sample Homepage!</title>
</head>
<body style="font-family: helvetica;">

  <br/>
  <br/>

  <h1 style="color:red">Welcome to my website!</h1>

  <p style="color:blue">Thanks for visiting my page! I'm a student and a pretty cool person. I once fought off a bear to save a crying baby! I also have purple hair.</p>

  <p style="color:blue">Here are some links to things:
  <ul style="color:green">
    <li><a href="http://www.princeton.edu/website.html">Here</a>'s the tutorial I got this from!
    <li>Look at this cute <a href="adorable_kitten.jpg">kitten</a>.
  </ul>

  <p style="color:blue">And here is a puppy!</p>

  <img style="height:300px" src="http://i.imgur.com/qF1qfC9.jpg">

</body>
</html>

    There you go -- you should have the beginning of a website! If you have any questions or want help making it prettier, feel free to email!

Troubleshooting

These are common problems I've seen encountered. I'll be adding to this as problems are reported, so feel free to contact me if you have any issues!

  • I'm not able to connect to my public_html folder or H: drive.

    You may be looking at something like this:

    Double check that you have set up your VPN if you are not on Princeton's wifi (Mac|Windows). Make sure your VPN is currently connected and has not disconnected. Reconnect and make sure you type everything exactly -- follow OIT's instructions (Mac|Windows).

    If the problem persists, consider using ssh to connect. Otherwise, feel free to contact me.

  • It gives me a 403 Error when I try to open the page -- the page tells me I don't have permission to view it.

    You may be looking at something like this:

    Somehow, the permissions for your website became messed up, and your folder is preventing anyone else from reading it. Luckily, we can fix this!

    First, make sure your Unix account is activated here. Next, we need to connect to your H: drive through what's called ssh.

    If you have a Mac or Linux, open the app Terminal. If you have Windows, you will need to ssh using PuTTY. Then, type ssh netid@arizona.princeton.edu. Type y if you get a question, and then type your netid password when prompted.

    Now, we should be looking at our folder -- something like this (type ls to see the files):

    Next, type cd /PUBLIC_HTML/user2a/. This is switching to the actual folder which OIT connects to the internet (warning: there are about 10,000 folders here -- do not type 'ls' or your terminal may freeze!).

    Next, type chmod -R 755 your_netid. This resets the permissions for your folder properly -- this should fix the problem! Close your window to end the connection.

    If your problem was not fixed, feel free to contact me.
